PROPS DRILLING é um problema onde os dados precisam ser passados através de vários niveis(fluxograma) de componentes intermidiarios, mesmo quando esses dados são necessarios apenas em um componente, resultando em código mais complexo e propenso a erros, pois é necessario porpagar os props por varios niveis. Uma forma de resolver esse problema e utilizando Contextos pois permite compartilhar dados para componente especiificos

Um bom exemplo é de uma questão que foi feita em sala onde se criava 4 componentes chamados funções(a,b,c,d). E cada um retornava um jsx imprindo seu nome, foi criado uma variavel constante chamada cores com valor "red" e queria passa esta variavel via props para os outros componentes para utilizarem na tag (h1 style={{backgroundColor:cores.bkgD}}) porem era muito verboso(não que contextos não seja para este exemplo). Para solucionar este problema foi criado um arquivo MeuContexto com uma função constante Cor, utiliza-se "provider" para prover aos componentes hierarquicamente abaixo de funçãoA e passava um valor, importava o contexto para os componetes especificos e utilizava a constante Cor para acessar a variavel "bkg(a,b,c,d)" que representava varias cores para aplicar na tag do componente
